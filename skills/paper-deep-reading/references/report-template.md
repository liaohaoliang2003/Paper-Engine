# Report Template

用途：生成单篇论文 7 章主干报告；第 4 章按原文一级/二级标题展开并填充固定槽位。

## 固定一级章节（不得改名/改序）

# 1、简短总结
- 用 1-3 段给出论文目标、核心方法、关键结果和一句价值判断。
- 明确区分：`原文陈述` 与 `推断`。
- 关键结论后独立行给出：
  - `证据锚点：...`

# 2、核心贡献与新颖性
- 列出 3-6 条贡献。
- 每条贡献补充“相对已有工作的新意边界”。
- 每条贡献后独立行给出：
  - `证据锚点：...`

# 3、针对的问题
- 说明论文试图解决的问题、前提假设、问题边界。
- 说明问题是否偏方法创新/数据构建/评测协议。

# 4、分段详解
- 必须按论文原有一级/二级标题顺序展开。
- 若原文无显式二级标题：按自然子段落构造等价二级，并标注“等价二级”。
- 每个二级小节至少 200-350 字；复杂小节可更长。
- 每个二级小节必须包含以下四个槽位：

## <原文二级标题A>
小节总结：...

深入分析：...

术语解释：...（至少 1 个关键术语；术语写法 `English Term（中文翻译）`）

示例或类比：...（若不适用，写“不适用原因”）

证据锚点：章节/段落/表格/公式定位...

## <原文二级标题B>
小节总结：...

深入分析：...

术语解释：...

示例或类比：...

证据锚点：...

# 5、局限性与风险评估
- 说明失效场景、假设依赖、潜在偏差和证据薄弱点。
- 对关键风险给出置信等级（高/中/低）。

# 6、后续可能的创新点和改进点
- 给出 3-6 条可执行改进方向。
- 每条建议应包含“为何可行 + 预期收益 + 潜在代价”。

# 7、复现计划
- 最小可复现清单：数据、环境、关键超参、训练流程、评测协议。
- 风险与排障优先级：最可能失败的 3 个点。
- 产出文件：`{paper_stem}_report.md` 与 `{paper_stem}_report.html`。

## 可扩展章节（可选）
- 只能放在第 7 章之后。
- 允许名称：`证据-结论对照表`、`关键术语表`、`附录`。

## 强约束提示
- `术语解释：` 与 `证据锚点：` 必须独立成行（用于 HTML 手风琴识别）。
- 信息不足时明确写：`信息不足/不确定性`，禁止无证据补全。
