# Quality Checklist

用途：给写作流程与 `scripts/validate_report.ps1` 提供统一门禁编号。

## Hard Checks（失败即不合格）

- `H001` Markdown 文件存在。
- `H002` HTML 文件存在。
- `H101` 前 7 个一级章节顺序固定且标题匹配。
- `H102` 第 4 章包含二级标题结构（按原文展开）。
- `H103` 第 4 章包含四槽位关键词（小节总结/深入分析/术语解释：/示例或类比）。
- `H104` 存在 `证据锚点：` 独立行。
- `H105` 存在 `术语解释：` 独立行。
- `H106` 文本长度达到最小阈值（默认 >= 4500 字符）。
- `H107` 术语双语格式可检测通过（英文术语应含 `（中文）`）。
- `H201` HTML 包含原论文链接（相对路径）。
- `H202` HTML 包含手风琴结构（trigger + panel）。
- `H203` HTML 包含可访问性交互（`aria-expanded` + Enter/Space）。
- `H204` HTML 手风琴无空面板。

## Soft Checks（告警项）

- `S301` 扩展章节仅位于第 7 章之后。
- `S302` 术语解释尽量避免重复讲解已充分解释术语。
- `S303` 每章至少一个可理解例子（含类比）。

## 使用方式
- 草稿阶段：可先看 soft 告警。
- 交付阶段：hard 必须全通过。
- 默认策略：hard 失败即退出（非 0）。
